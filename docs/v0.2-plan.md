# v0.2 Plan

## Objective

`v0.2` の目的は、`v0.1` の `IPP -> cell aggregate` 型の可視化パイプラインを、より研究用途に耐える `regional VTEC field estimation` へ寄せることである。

## Why v0.1 is not enough

`v0.1` の前処理は、単局から複数局への拡張と Web 可視化の導線としては十分だが、日本全域の連続 TEC 場を出すには不足がある。

不足点:

- GPS 優先で multi-GNSS が弱い
- LLI / SNR / elevation を十分に使っていない
- receiver bias / satellite bias / arc bias の分離が弱い
- 格子セルに観測を落とすだけで、全国格子全体を推定していない
- quiet-day baseline が未実装

## Keep / Change

### Keep

- `RINGO -> CSV`
- ingest
- geometry-free
- cycle slip / arc segmentation
- station metadata
- `az/el` を使った IPP 補助
- FastAPI / frontend / Makefile / tests

### Change

- 観測コード選択を multi-GNSS 前提へ拡張
- quality filter を強化
- bias 推定を明示化
- VTEC grid を unknown とする全国場推定へ変更
- anomaly baseline を quiet-day ベースへ変更

## Planned v0.2 modules

### 1. Observation quality control

追加項目:

- LLI 利用
- SNR threshold
- elevation mask
- outlier rejection
- constellation 別の観測コード管理

候補ファイル:

- `backend/app/processing/quality_control.py`

### 2. Bias estimation

目的:

- receiver differential code bias
- satellite differential code bias
- arc bias

を明示的に扱う。

候補ファイル:

- `backend/app/processing/bias.py`

### 3. Multi-GNSS ingest

対象:

- GPS
- GLONASS
- Galileo
- QZSS

候補ファイル:

- `backend/app/config/gnss.py`
- `backend/app/processing/ringo_ingest.py`

### 4. Regional VTEC estimation

現在の `IPP -> grid aggregate` を置き換える。

方向:

- `0.5° x 0.5°`
- `10 min` or `15 min`
- 全国格子の VTEC を unknown にする
- LOS 観測から正則化付き最小二乗で解く

候補ファイル:

- `backend/app/processing/regional_vtec.py`

### 5. Baseline / anomaly

変更:

- quiet-day baseline
- local time 対応 baseline
- `z-score` と `detrended TEC` の両立

候補ファイル:

- `backend/app/processing/anomaly.py`

## Proposed implementation order

1. multi-GNSS ingest
2. quality control 強化
3. bias estimation
4. regional VTEC estimation
5. quiet-day baseline
6. frontend の mode 切替

## Success criteria

`v0.2` では次を満たすことを目標にする。

- 日本全域で欠測が減る
- 同一時刻でより連続的な VTEC 分布が得られる
- multi-GNSS により LOS 密度が上がる
- anomaly が簡易 baseline ではなく quiet-day 基準になる
